Glossary

Member: a ChocAn customer that pays a monthly fee in order to receive services provided by healthcare professionals affiliated with ChocAn; each member has a unique 9-digit number

Member Status: a member's status is set to suspended if they have not paid membership fees for a least a month

Provider: a healthcare professional who provides ChocAn services to members

Service Code: a unique code that corresponds to a specific service

Provider Directory: list of services containing their corresponding service codes and fees; can be sent to the provider as an email attachment

Member Report: a weekly report sent to members that contains their personal information, the services they received that week, the date of the services, and the name of the provider

Provider Report: a weekly report sent to providers that contains their personal information, the services they provided for the week, and the fee to be paid by ChocAn

Summary Report: a report that lists all providers to be paid for the week, along with the number of consultations they provided and the fee to be paid to them; the report also lists the total number of providers/consultations, and the overall fee total

Manager: a ChocAn employee responsible for handling reports

Operator: a ChocAn employee responsible for maintaining member and provider records

Acme Accounting Services: third party organization responsible for various ChocAn financial procedures

Use Case descriptions

List:

  • Validate Member
  • Validate Provider
  • Record Service
  • Request Provider Directory
  • Generate Provider Report for Provider
  • Generate Member Report
  • Generate file for EFT data
  • Generate Overall Provider Summary Report for Manager
  • Manage People Directory
  • Add/Delete Member extends Manage People Directory
  • Update Members' Records extends Manage People Directory
  • Add/Delete Provider extends Manage People Directory
  • Update Provider Record extends Manage People Directory
  • Run Main Accounting Procedure

Description:

  • Use case: Validate Member
  • Context: Validation that a customer is a current member of ChocAn.
  • Actors: Provider, ChocAn Data Center
  • Main Success Scenario:
    1. Member provides ChocAn membership card to provider
    2. Provider slides card through card reader on the terminal
    3. Terminal dials ChocAn data center
    4. ChocAn data center verifies member number
    5. Validated appears on terminal screen
    6. Card is ejected
  • Extensions:
    • 5a. Verification fails
      • 5a1. Invalid number appears on terminal screen if number is invalid
        • 5a1a. Use case continues at step 6
      • 5a2. Member suspended appears on terminal screen if member owes fees
        • 5a2a. Use case continues at step 6
  • Use case: Validate Provider
  • Context: Provider enters provider number to designate the service being done as by a specific provider.
  • Actors: ChocAn Data Center, Provider
  • Main Success Scenario:
    1. Provider switches on their terminal.
    2. Provider enters their provider number into the terminal.
    3. Terminal dials the number to the ChocAn data center and validates the number.
    4. The terminal accepts the number and allows the provider to use the software.
  • Extensions:
    • 3a. Provider number fails to validate
      • 3a1. Validation failure is printed to screen
      • 3a2. Use case returns to step 2
  • Use case: Record Service
  • Context: Records data about the service provided to disk
  • Actors: Provider, ChocAn Data Center
  • Main Success Scenario:
    1. The provider validates the member.
    2. Provider keys in date the service was provided (in MM-DD-YYYY)
    3. Provider references provider directory to find six-digit service code which corresponds to a specific service provided
    4. Provider keys in the service code to the terminal
    5. The software allows the provider to validate the code was correct by displaying what the corresponding service is on the screen
    6. The provider asks for validation that the service is correct
    7. The provider can offer comments on the service provided
    8. The software writes: Current date and time (in MM-DD-YYY HH:MM:SS), date service was provided (in MM-DD-YYYY), 9 digit provider number, 9 digit member number, 6 digit service code, and an optional 100 character comment on the service.
    9. Software checks service code against its match in the ChocAn data center
    10. Data center returns fee for respective service
    11. Following this, the software prints the fee to be paid for the service on the terminal screen.
    12. The provider validates the transaction by entering the current date and time, the service provided date, member name and number, the service code, and the fee value.
  • Extensions:
    • 1a. Verification fails
      • 1a1. Error code is printed to screen
      • 1a2. Use case returns to step 1
    • 5a. Number does not exist in the system
      • 5a1. Error code is printed to screen
      • 5a2. Use case returns to step 4
    • 12a. Validation fails
      • 12a1. Validation error printed to screen
      • 12a2. Use case returns to step 12
  • Use case: Request Provider Directory
  • Context: Provider can request an alphabetically ordered list of service names, codes, and fees.
  • Actors: Provider
  • Main Success Scenario:
    1. Provider accesses the terminal
    2. Provider queries the ChocAn software for a provider directory
    3. The software e-mails the Provider Directory as an attachment
  • Extensions:
    • N/A
  • Use case: Generate Provider Report for Provider
  • Context: System automatically generates a report for that provider’s services rendered during the week.
  • Actors: ChocAn Data Center, Timer
  • Main Success Scenario:
    1. At the end of the week, the data center generates a report containing: Provider name, provider number, provider street address, provider city, provider state, provider ZIP code, details on each service provided that week, including: Date of service, data and time data were received by computer, member name, member number, service code, fee to be paid. Additionally, the report includes the total number of consultations with members that week, as well as the total fee for that week.
    2. The report is then sent to the respective ChocAn provider.
  • Extensions:
    • N/A
  • Use case: Generate Member Report
  • Context: Generates member report detailing the services obtained by the respective member.
  • Actors: Timer, ChocAn Data Center
  • Main Success Scenario:
    1. At midnight Friday, the ChocAn Data Center generates a member report containing: member name, member number, member street address, member city, member state, member ZIP code, details of each service provided including: date of service, provider name, and service name.
    2. The report is then sent as an e-mail attachment to that respective member
  • Extensions:
    • N/A
  • Use case: Read/Collect Data from Providers' Service Reports
  • Context: Information is collected and compiled from each service report
  • Actors: Terminal, Provider, ChocAn Data Center
  • Main Success Scenario:
    1. The providers’ service reports are compiled by the data center
    2. Information is received from the providers’ forms and the number of services performed based on date
    3. The providers’ service reports are compiled by the data center
    4. Total fee charged by the providers is retrieved
  • Extensions:
    1. Information is incomplete or improperly entered
  • Use case: Generate file for EFT data
  • Context: Although the financial component is mainly handled by ACME, the EFT file must be compiled by the software first.
  • Actors: ChocAn Data Center
  • Main Success Scenario:
    1. Individual provider information is compiled
    2. Retrieve provider name
    3. Retrieve provider number
    4. Retrieve amount to be transferred
    5. Write data to disk
  • Extensions:
    • 1a. Disk is corrupted
    • 1a1. Disk error printed to screen
    • 1a2. Use case returns to step 2
  • Use case: Generate Overall Provider Summary Report for Manager
  • Context: The manager receives a summary report for accounts payable. It contains extensive data about the providers.
  • Actors: Manager, ChocAn Data Center
  • Main Success Scenario:
    1. Data center compiles summary based on: Provider to be paid, Number of consultations by provider, Provider’s total fee, Total number of providers, Total number of consultations, Overall fee total
    2. Summary report is then provided to the manager so that the manager may deliberate which providers to pay for which services were rendered.
  • Extensions:
    • N/A
  • Use case: Manage People Directory
  • Context: Adds, deletes, or updates any persons (members or providers) within the ChocAn directory.
  • Actors: Operator
  • Main Success Scenario:
    1. Operator uses the ChocAn software to modify data on a person present in the ChocAn directory.
    2. If the operator adds or deletes a member, then the system accesses the functionality for adding and deleting members.
    3. If the operator adds or deletes a provider, then the system accesses the functionality for adding and deleting providers.
    4. If the operator updates a member, then the system accesses the functionality for modifying the member data.
    5. If the operator updates a provider, then the system accesses the functionality for modifying the provider data.
  • Extensions:
    • 4a. Member does not exist
    • 4a1. Error printed to screen.
    • 4a1. Error printed to screen
    • 5a. Provider does not exist
    • 5a1. Error printed to screen
    • 5a2. Use case returns to step 1
  • Use case: Add/Delete Member extends Manage People Directory
  • Context: Operators are able to add or delete members through the ChocAn Data Center’s interactive mode during the day.
  • Actors: Operators, ChocAn Data Center
  • Main Success Scenario:
    1. Operator logs into ChocAn Data Center
    2. Operator logs into ChocAn Data Center
    3. Perform operation on membership statuses
    4. Operator adds new member
    5. Operator deletes current member
  • Extensions:
    • 2a. Member to delete does not exist
    • 2a1. Error printed to screen
    • 2a2. Use case returns to step 2
  • Use case: Update Members’ Records extends Manage People Directory
  • Context: Operators are able to update member details through the ChocAn Data Center’s interactive mode during the day.
  • Actors: Operators, ChocAn Data Center
  • Main Success Scenario:
    1. Operator logs into ChocAn Data Center
    2. Operator enters member credentials
    3. Perform update on membership details
  • Extensions:
    • 2a. Member does not exist
    • 2a1. Error printed to screen
    • 2a2. Use case returns to step 2
  • Use case: Add/Delete Provider extends Manage People Directory
  • Context: Provider records are added or deleted as provider options change.
  • Actors: Provider, Operator, Terminal, ChocAn Data Center
  • Main Success Scenario:
    1. Operator logs into ChocAn Data Center
    2. Operator enters provider credentials
    3. Perform operation on provider statuses
    4. Operator adds new provider
    5. Operator deletes current provider
  • Extensions:
    • 2a. Provider does not exist
    • 2a1. Error printed to screen
    • 2a2. Use case returns to step 2
  • Use case: Update Provider Record extends Manage People Directory
  • Context: Provider records are updated as provider options change.
  • Actors: Provider, Operator, ChocAn Data Center
  • Main Success Scenario:
    1. Operator logs into ChocAn Data Center
    2. Operator enters provider credentials
    3. Perform update on provider details
  • Extensions:
    • 2a. Provider does not exist
    • 2a1. Error printed to screen
    • 2a2. Use case returns to step 2
  • Use case: Run Main Accounting Procedure includes Generate Member Report, Generate File for EFT Data, Generate Overall Provider Report Summary for Manager, and Generate Provider Report
  • Context: Generates all the reports which will be sent out at the end of the week.
  • Actors: Timer, ChocAn Data Center, Member, Provider, Manager
  • Main Success Scenario:
    1. The timer recognizes that the end of the week has commenced.
    2. The software system, pulling data from the ChocAn data center, generates the following reports: a. Individual Member Service Report for each Member, b. Individual Services Rendered Report for each Provider, c. Overall Provider Summary Report for Manager, d. EFT data
    3. The system then prints and e-mails reports to their respective intended recipients.
  • Extensions:
    • N/A
